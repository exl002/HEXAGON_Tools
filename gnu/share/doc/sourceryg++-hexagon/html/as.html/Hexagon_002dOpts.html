9.11.1 Hexagon Options

The Qualcomm Hexagon version of as has a few machine dependent options:

-G size
Set the maximum size of objects in the small-data area, accessible via the GP register, to size (default is 8).


-march=version
Generate code for the specified Hexagon version. The following version names are valid: v2, v3, v4, and v5.


-mcpu=version
Equivalent to -march=version.


-mv2
-mv3
-mv4
-mv5
Generate code for the given Hexagon version. Equivalent to the corresponding -march option.


-mno-dealloc_return
Disable use of the dealloc_return instruction in Hexagon V4 and later.


-mno-extender
Disable the use of constant extenders in Hexagon V4 and V5.


-mfalign-info
Report statistics for the effects of the directive .falign.


-mfalign-warn
Warn when the directive .falign results in padding instructions being added.


-mno-jumps
Disable automatic extension of some branch instructions in Hexagon V4 and V5.


-mno-jumps-long
Disable automatic extension of long (non-paired) branch instructions in Hexagon V4 and V5.


-mno-pairing
Disable pairing of packet instructions in Hexagon V4 and V5.


-mno-pairing-branch
Disable pairing of branch instructions in Hexagon V4 and V5.


-mno-pairing-duplex
Disable pairing to duplex instructions in Hexagon V4 and V5.


-mpairing-info
Report about instruction pairing in Hexagon V4 and V5.


-msort-sda
Sort objects in the small-data area in ascending order by their access size. -mno-sort-sda disables this default behavior.
