4.5 ld and the Hexagon family

The following Hexagon-specific command-line options are supported:

--march=architecture
Link for the specified Hexagon architecture. By default, the linker uses the architecture of input objects and criteria such as ABI and binary compatibility to determine the architecture of the output object. By using this option, it is possible to guarantee the architecture of the output object. The following architecture names are valid: ‘v2’, ‘v3’, ‘v4’, and ‘v5’.


--mcpu=architecture
Equivalent to ‘--march=architecture’.


-mv2
-mv3
-mv4
-mv5
Link for the specified Hexagon architecture. Equivalent to the corresponding ‘--march=’ option.


--tcm
Use the TCM sections and those for other memory buses. (not available in all emulations).


--trampolines
--no-trampolines
Control addition of trampolines to certain types of branches when the target is beyond the reach of the instruction offset. Trampolines are appended to the input section that contains the branch, so the end of the section must be within its range in order to add a trampoline successfully. Trampolines are code stubs that use the largest available branch instruction to reach the target. Therefore, they are not neutral in terms of performance and code size. The default is ‘--trampolines’.
